### 浏览器自动化

## [opencli](https://github.com/jackwener/opencli)

<RepoMeta
  url="https://github.com/jackwener/opencli"
  stars={21698}
  updated="2026-05-18"
  lang="JavaScript"
  license="Apache-2.0"
  tags={['cli-hub', 'browser-automation', 'ai-agent']}
/>

**把任意网站/工具变成 CLI + 让 AI agent 驱动你已登录的浏览器** — 通用 CLI Hub 与 AI-native runtime。三种自动化合一：① 内置 100+ 站点适配器（B站/知乎/小红书/Reddit/HackerNews/Twitter 等）；② 让 AI agent 通过你已登录的 Chrome 实时操作任意网站（导航/填表/点击/抽取）；③ 用 `opencli browser` + `opencli-adapter-author` skill 端到端写新适配器。也能当 CLI Hub 纳管 gh/docker/wx 等本地工具，并通过 CDP 控制 Cursor/Codex/ChatGPT 等 Electron 桌面应用。

**能做**：
- 实时浏览器自动化 —— 凭证留在浏览器，运行时**零 LLM 成本**（确定性输出、不烧 token）
- 100+ 内置适配器 + 外部 CLI passthrough + Electron 桌面应用控制
- 多 Chrome profile 路由（`--profile`）；AI agent 经 `npx skills add jackwener/opencli` 装配套 skill 使用

**能力边界 ⚠️**：
- **强依赖 Browser Bridge 浏览器扩展 + Chrome/Chromium** —— 没装扩展，浏览器类命令全部失效
- 需 Node.js ≥ 21
- 依赖你**已登录**的浏览器会话 —— 它不替你登录、不过验证码，凭证管理是你自己的事
- 网站改版会让适配器失效，需 `opencli-autofix` skill 修复
- 它是接口层 / runtime，不是爬虫框架本身；浏览器命令"设计给 AI agent 用，非手动跑"

### 测试 & UI 自动化

## [midscene](https://github.com/web-infra-dev/midscene)

<RepoMeta
  url="https://github.com/web-infra-dev/midscene"
  stars={13235}
  updated="2026-05-22"
  lang="TypeScript"
  license="MIT"
  tags={['ui-automation', 'ai-test', 'computer-use', 'mcp']}
/>

**一句话定位** — AI 驱动、视觉感知的全平台 UI 自动化工具，用自然语言描述目标即可让 Agent 规划并操作界面。Midscene 支持 Web、Android、iOS 以及任意可截图界面，可与 Playwright/Puppeteer、ADB、WebDriverAgent 和 MCP 集成，用于自动化测试、数据提取、UI 断言与跨端操作。

**亮点**：
- 纯视觉路线定位和操作 UI，适合 DOM 不稳定、Canvas、移动端和桌面界面场景
- 提供 `aiAssert()`、`aiLocate()`、`aiWaitFor()` 等开发者 API，以及可视化回放报告、Playground、Chrome 插件
- 支持 MCP 暴露原子 UI 操作，方便上层 Agent 调用，并可接入 UI-TARS、Qwen-VL、Gemini 等视觉模型

**注意**：⚠️ 这是自动化/测试与 computer-use 基础设施，需要配置模型、浏览器或移动端运行环境；复杂任务的稳定性取决于视觉模型和目标 UI 状态。
