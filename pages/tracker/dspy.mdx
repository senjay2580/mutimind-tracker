> [stanfordnlp/dspy](https://github.com/stanfordnlp/dspy) — Stanford NLP 出品。**Declarative Self-improving Python**。把"写 prompt"变成"写程序"——LLM 应用的编译器框架。

### 核心思想

不再手写 prompt 字符串，而是**声明式模块化代码**。框架自动编译/优化 prompt + few-shot demos，必要时还能 fine-tune 权重。同一份代码切换 GPT-4 / Claude / Llama / 本地模型只需重新 compile。

### 关键能力

| 能力 | 说明 |
|---|---|
| 模块化 | `Predict` / `ChainOfThought` / `ReAct` / `ProgramOfThought` 可组合模块 |
| Signature 签名 | `"input -> output"` 字符串声明任务，不写 prompt |
| Optimizer 优化器 | `MIPROv2` / `BootstrapFewShot` / `GEPA` 自动调 prompt 与 demo |
| 跨模型 | 同一份代码，切 LM 重新编译 |
| 指标驱动 | 给 metric 函数，框架自己迭代提升 |

### 应用场景

| 场景 | 收益 |
|---|---|
| RAG 问答系统 | 自动优化每一步 prompt + few-shot |
| 结构化信息抽取 | Signature 声明字段，保证输出格式 |
| 多跳推理 / Agent | `dspy.ReAct` 模块 + 自动优化思考链 |
| 文本分类 / 打标 | 编译后小模型≈大模型效果，省钱 |
| 代码生成 / 修复 | `ProgramOfThought` + metric 闭环 |
| 跨模型迁移 | 同代码换 LM 即可 |
| 企业评测体系 | 把 metric 喂 optimizer 自动迭代 |

**不适合**：一次性问答、没有 metric、创意类生成（写诗等无客观指标）

### 标准开发流程（7 步）

```
①定义任务 → ②写 Signature → ③组装 Module
     ↓
④准备 Trainset + Metric → ⑤选 Optimizer 编译
     ↓
⑥评估对比 → ⑦保存编译产物部署
```

### 最小代码骨架

```python
import dspy
from dspy.teleprompt import MIPROv2

# ② Signature：声明输入输出
class GenerateAnswer(dspy.Signature):
    """根据上下文回答问题，必须给出引用"""
    context: str = dspy.InputField()
    question: str = dspy.InputField()
    answer: str = dspy.OutputField()
    citations: list[str] = dspy.OutputField()

# ③ Module：组装程序
class RAG(dspy.Module):
    def __init__(self):
        self.retrieve = dspy.Retrieve(k=5)
        self.generate = dspy.ChainOfThought(GenerateAnswer)
    def forward(self, question):
        ctx = self.retrieve(question).passages
        return self.generate(context=ctx, question=question)

# ④ 数据 + Metric
trainset = [dspy.Example(question="...", answer="...").with_inputs("question"), ...]
def metric(ex, pred, trace=None):
    return ex.answer.lower() in pred.answer.lower()

# ⑤ 编译（核心魔法）
optimizer = MIPROv2(metric=metric, auto="medium")
compiled_rag = optimizer.compile(RAG(), trainset=trainset, valset=valset)

# ⑦ 保存部署
compiled_rag.save("rag_v1.json")
```

### 三个心智转变

1. **不手调 prompt** → 让 optimizer 调
2. **先写代码结构再想 prompt** → Signature 是接口，prompt 是实现细节
3. **没 metric 不用 DSPy** → 核心价值是「指标驱动迭代」
