**"零人工公司" Agent 操作系统** — Slogan 直接写「The open-source app everyone uses to manage agents at work」。**像管真实公司一样管 AI 员工**：组建 Agent 团队、设组织架构（CEO / 工程 / 销售 / 客服...）、分配目标、跟踪进度、看产出。Node.js + React 自托管，[paperclip.ing](https://paperclip.ing) 是官方 SaaS。**66k★ 短期内积累**，是 2026 上半年 agent 平台赛道最大黑马。

**典型搭配**：[OpenClaw](https://github.com/openclawai)（员工角色）+ Paperclip（公司组织层）—— Claw 当员工，Paperclip 当公司，把"一个开发者管 50 个 AI 员工"落到具体可执行的工程范式。

**亮点**：
- 自托管 TS 全栈，部署门槛低
- 组织架构 + Issue 跟踪 + 产出审计完整链
- 和 deer-flow / multica 等同代竞品比，**更偏"公司运营隐喻"而非"工程框架"**

**注意**：⚠️ 短期内爆涨的项目，工程稳定性 / 数据安全实践还在验证；商用前自己跑生产 ready 评估。

**开源 Agent 编排器（orchestrator）** — **是什么**：一个不绑定 IDE、不绑定单一 agent、不绑定封闭模型生态的可插拔编排器，`npx pikiclaw@latest` 一行起，Node 20+。**能干什么**：让创作者不碰代码、从一个控制台并行驱动「一群 agent」，通过四层架构自由组合 —— ① **Terminal 层**：Telegram / 飞书 / 微信 / Slack / Discord / 钉钉 / 企业微信 / Web Dashboard 全是平级入口，从最顺手的 IM 直接发指令；② **Agent 层**：Claude Code / Codex / Gemini / Hermes 经 driver registry + ACP 接入；③ **Model 层**：Claude / GPT / Gemini / DeepSeek / 豆包 / MiMo / MiniMax / OpenRouter 及任意 OpenAI 兼容代理任选；④ **Tool 层**：Skills / MCP / CLI（global × workspace 两级）。

**重点 & 特色**：
- **产品本体就是编排器** —— 刻意反「软件创作必须困在单个代码编辑器里」的范式，赌「下一代创作发生在 orchestrator 而非 IDE」
- **完全自举（self-bootstrapped）** —— pikiclaw 自身就是用 pikiclaw 造出来的，把「用 agent 群造软件」当作自证
- **IM 即终端** —— 把 agent swarm 指挥台搬进你已在用的聊天软件，新终端即插即用

**注意**：⚠️ 仅 268★、2026-05 起步的早期项目，driver registry / ACP 生态与工程稳定性仍处早期，生产用前自评。
